# Yandex Maps Geocoder API

REST API для поиска географических объектов (улиц, городов, районов, станций метро, домов) через Яндекс.Геокодер.

## 1. Назначение сервиса

Сервис предоставляет REST API для поиска ближайших географических объектов по координатам. Основные функции:
- Поиск улиц, городов, районов, станций метро, домов и достопримечательностей
- Возврат информации об объекте: название, адрес, тип, расстояние до указанных координат
- Интеграция с Яндекс.Геокодер API

В рамках микросервисной архитектуры данный сервис может использоваться как:
- Самостоятельный микросервис геоданныхй
- Базовый сервис для построения маршрутов и навигации

## 2. Архитектура и зависимости

### Технологии и фреймворки
- **Python 3.11+** — язык программирования
- **FastAPI** — веб-фреймворк для построения REST API
- **Uvicorn** — ASGI сервер
- **HTTPX** — асинхронный HTTP клиент для запросов к внешним API
- **Pydantic** — валидация данных и сериализация
- **python-dotenv** — управление переменными окружения

### Инструменты разработки
- **Black** — форматирование кода
- **isort** — сортировка импортов
- **flake8** — линтер
- **pytest** — тестирование
- **pytest-asyncio** — поддержка асинхронных тестов
- **pytest-cov** — покрытие кода тестами

### Взаимодействие с внешними сервисами
| Сервис | Назначение | Тип взаимодействия |
|--------|------------|-------------------|
| Яндекс.Геокодер API | Поиск географических объектов | HTTP запросы (REST) |

Сервис не взаимодействует с другими микросервисами, является самодостаточным.

### Используемые внешние сервисы
- **Яндекс.Геокодер API** (https://geocode-maps.yandex.ru/1.x/) — для получения данных о географических объектах

## 3. Способы запуска сервиса

### Переменные окружения (.env)

Создайте файл `.env` в корне проекта:
```env
YANDEX_API_KEY=<ваш_ключ_яндекс_геокодера>
```
### Локальный запуск (без Docker)

1. Клонирование репозитория 
```bash
git clone <ваш-репозиторий>
cd yandex-maps-nearby-api
```

2. Установка зависимостей
```bash
pip install -r requirements.txt
```

3. Запуск сервера
```bash
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

4. Проверка работы
- Откройте в браузере: http://localhost:8000/docs

## 4. API документация

### Интерактивная документация

- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

### Основные эндпоинты

#### GET /
Корневой эндпоинт для проверки работоспособности.

**Ответ:**
```json
{
  "message": "Yandex Maps Nearby API",
  "docs": "/docs",
  "redoc": "/redoc"
}
```

#### GET /api/v1/nearby
Поиск ближайших географических объектов.

**Параметры запроса:**

| Параметр | Тип | Обязательный | Описание | Пример |
|----------|-----|--------------|----------|--------|
| `lat` | float | ✅ | Широта | 55.7558 |
| `lon` | float | ✅ | Долгота | 37.6173 |
| `type` | string | ❌ | Тип объекта (по умолчанию "улица") | улица, город, район, метро, дом |
| `limit` | int | ❌ | Количество результатов (1-50, по умолчанию 10) | 5 |

**Поддерживаемые типы объектов:**
- `улица` — поиск улиц (Временно не работает)
- `город` — поиск городов
- `район` — поиск районов
- `метро` — поиск станций метро
- `дом` — поиск домов (Временно не работает)
- `достопримечательность` — поиск достопримечательностей (Временно не работает)

**Пример запроса:**
```
GET /api/v1/nearby?lat=55.7558&lon=37.6173&type=улица&limit=5
```

**Пример ответа:**
```json
[
  {
    "name": "Тверская улица",
    "address": "Москва",
    "type": "улица",
    "distance": 125.3,
    "coordinates": {
      "lat": 55.7575,
      "lon": 37.6108
    }
  }
]
```
